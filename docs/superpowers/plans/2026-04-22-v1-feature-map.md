## What "one feature" means here

Each feature plan is **one focused PR** that ships working, testable software. A feature should:
- Land behind a green CI build
- Not break any existing functionality (the `/vault` route must still render)
- Include its own tests for the new behavior it introduces
- Not depend on a later feature to "complete" it

If any `v1-fN` can't meet those criteria, it needs to be split further before being written.
